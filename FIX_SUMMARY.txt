DEPLOYMENT ERROR FIX - Quick Reference

Problem

Your deploy-mcp.bat script failed with "ERROR: Dependency Installation Failed" when attempting to install Python packages in Windows 10 VM.

Root Cause

The original script was running pip install with quiet mode and suppressed error output, making it impossible to diagnose failures. Additionally, there were no pre-flight checks to validate the environment before attempting installation.

Solution Applied

The deployment script has been completely rewritten with:

1. Verbose installation output (no more quiet mode)
2. Pre-flight environment validation
3. Individual error handling for each package group
4. Post-installation verification
5. Better error messages with actionable solutions

New Tools Created

1. install-dependencies-manual.bat
   Run this to install packages one by one with full visibility.
   Use this as your first attempt to fix the issue.

2. diagnose-environment.bat
   Run this to identify exactly what is wrong with your environment.
   Tests 10 different aspects (Python, pip, network, permissions, etc.)

3. TROUBLESHOOTING.txt
   Complete troubleshooting guide covering all common scenarios.

What To Do Now

Step 1: Try the manual installer

   install-dependencies-manual.bat

   This will install each package individually and show exactly where it fails.
   If this succeeds, run deploy-mcp.bat again and it will skip to configuration.

Step 2: If manual installer fails, run diagnostics

   diagnose-environment.bat

   This will show you exactly what is wrong:
   - Python installation issues
   - pip problems
   - Network connectivity
   - Firewall blocking PyPI
   - Permission issues
   - Antivirus interference

Step 3: Fix the identified issue

   Follow the solutions provided by the diagnostic tool.
   Common fixes:
   - Run as Administrator
   - Configure corporate proxy settings
   - Enable internet access
   - Disable antivirus temporarily
   - Reinstall Python (avoid Microsoft Store version)

Step 4: Run deploy-mcp.bat again

   After fixing the issue, the full deployment should work.

Expected Issues in VM Environments

No Internet Connection
   Solution: Ensure VM has network access
   Test: ping pypi.org

Corporate Proxy
   Solution: Set environment variables:
   set HTTP_PROXY=http://proxy:port
   set HTTPS_PROXY=http://proxy:port

Insufficient Permissions
   Solution: Right-click batch file > Run as administrator

Antivirus Blocking
   Solution: Temporarily disable or add Python to exclusions

Python Not in PATH
   Solution: Reinstall Python with "Add to PATH" checked
